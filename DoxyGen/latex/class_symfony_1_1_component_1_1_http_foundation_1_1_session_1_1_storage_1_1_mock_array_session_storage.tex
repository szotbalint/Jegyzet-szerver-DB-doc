\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_symfony_1_1_component_1_1_http_foundation_1_1_session_1_1_storage_1_1_mock_array_session_storage}{Mock\+Array\+Session\+Storage}} mocks the session for unit tests.

No PHP session is actually started since a session can be initialized and shutdown only once per PHP execution cycle.

When doing functional testing, you should use \mbox{\hyperlink{class_symfony_1_1_component_1_1_http_foundation_1_1_session_1_1_storage_1_1_mock_file_session_storage}{Mock\+File\+Session\+Storage}} instead.

{\footnotesize\ttfamily regenerate (\begin{DoxyParamCaption}\item[{}]{\$destroy = {\ttfamily false},  }\item[{}]{\$lifetime = {\ttfamily null} }\end{DoxyParamCaption})}

\{Regenerates id that represents this storage.

This method must invoke session\+\_\+regenerate\+\_\+id(\$destroy) unless this interface is used for a storage object designed for unit or functional testing where a real PHP session would interfere with testing.

Note regenerate+destroy should not clear the session data in memory only delete the session data from persistent storage.

Care\+: When regenerating the session ID no locking is involved in PHP\textquotesingle{}s session design. See \href{https://bugs.php.net/61470}{\texttt{ https\+://bugs.\+php.\+net/61470}} for a discussion. So you must make sure the regenerated session is saved BEFORE sending the headers with the new ID. Symfony\textquotesingle{}s Http\+Kernel offers a listener for this. See Symfony\textbackslash{}\+Component\textbackslash{}\+Http\+Kernel\textbackslash{}\+Event\+Listener\textbackslash{}\+Save\+Session\+Listener. Otherwise session data could get lost again for concurrent requests with the new ID. One result could be that you get logged out after just logging in.

bool & {\em \$destroy} & Destroy session when regenerating? \\
\hline
int & {\em \$lifetime} & Sets the cookie lifetime for the session cookie. A null value will leave the system settings unchanged, 0 sets the cookie to expire with browser session. Time is in seconds, and is not a Unix timestamp.\\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
bool True if session regenerated, false if error
