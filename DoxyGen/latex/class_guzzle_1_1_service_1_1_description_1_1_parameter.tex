Create a new \mbox{\hyperlink{class_guzzle_1_1_service_1_1_description_1_1_parameter}{Parameter}} using an associative array of data. The array can contain the following information\+:
\begin{DoxyItemize}
\item name\+: (string) Unique name of the parameter
\item type\+: (string$\vert$array) Type of variable (string, number, integer, boolean, object, array, numeric, null, any). Types are using for validation and determining the structure of a parameter. You can use a union type by providing an array of simple types. If one of the union types matches the provided value, then the value is valid.
\item instance\+Of\+: (string) When the type is an object, you can specify the class that the object must implement
\item required\+: (bool) Whether or not the parameter is required
\item default\+: (mixed) Default value to use if no value is supplied
\item static\+: (bool) Set to true to specify that the parameter value cannot be changed from the default
\item description\+: (string) Documentation of the parameter
\item location\+: (string) The location of a request used to apply a parameter. Custom locations can be registered with a command, but the defaults are uri, query, header, body, json, xml, post\+Field, post\+File.
\item sent\+As\+: (string) Specifies how the data being modeled is sent over the wire. For example, you may wish to include certain headers in a response model that have a normalized casing of Foo\+Bar, but the actual header is x-\/foo-\/bar. In this case, sent\+As would be set to x-\/foo-\/bar.
\item filters\+: (array) Array of static method names to to run a parameter value through. Each value in the array must be a string containing the full class path to a static method or an array of complex filter information. You can specify static methods of classes using the full namespace class name followed by \textquotesingle{}\+::\textquotesingle{} (e.\+g. Foo\textbackslash{}\+Bar\+::baz()). Some filters require arguments in order to properly filter a value. For complex filters, use a hash containing a \textquotesingle{}method\textquotesingle{} key pointing to a static method, and an \textquotesingle{}args\textquotesingle{} key containing an array of positional arguments to pass to the method. Arguments can contain keywords that are replaced when filtering a value\+: \textquotesingle{}@value\textquotesingle{} is replaced with the value being validated, \textquotesingle{}@api\textquotesingle{} is replaced with the \mbox{\hyperlink{class_guzzle_1_1_service_1_1_description_1_1_parameter}{Parameter}} object.
\item properties\+: When the type is an object, you can specify nested parameters
\item additional\+Properties\+: (array) This attribute defines a schema for all properties that are not explicitly defined in an object type definition. If specified, the value MUST be a schema or a boolean. If false is provided, no additional properties are allowed beyond the properties defined in the schema. The default value is an empty schema which allows any value for additional properties.
\item items\+: This attribute defines the allowed items in an instance array, and MUST be a schema or an array of schemas. The default value is an empty schema which allows any value for items in the instance array. When this attribute value is a schema and the instance value is an array, then all the items in the array MUST be valid according to the schema.
\item pattern\+: When the type is a string, you can specify the regex pattern that a value must match
\item enum\+: When the type is a string, you can specify a list of acceptable values
\item min\+Items\+: (int) Minimum number of items allowed in an array
\item max\+Items\+: (int) Maximum number of items allowed in an array
\item min\+Length\+: (int) Minimum length of a string
\item max\+Length\+: (int) Maximum length of a string
\item minimum\+: (int) Minimum value of an integer
\item maximum\+: (int) Maximum value of an integer
\item data\+: (array) Any additional custom data to use when serializing, validating, etc
\item format\+: (string) Format used to coax a value into the correct format when serializing or unserializing. You may specify either an array of filters OR a format, but not both. Supported values\+: date-\/time, date, time, timestamp, date-\/time-\/http
\item \$ref\+: (string) String referencing a service description model. The parameter is replaced by the schema contained in the model.
\end{DoxyItemize}
