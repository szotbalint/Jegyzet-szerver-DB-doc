\doxysubsection{Detailed Description}
\mbox{\hyperlink{namespace_omnipay_1_1_stripe}{Stripe}} Transfer Request.

To send funds from your \mbox{\hyperlink{namespace_omnipay_1_1_stripe}{Stripe}} account to a connected account, you create a new transfer object. Your \mbox{\hyperlink{namespace_omnipay_1_1_stripe}{Stripe}} balance must be able to cover the transfer amount, or you\textquotesingle{}ll receive an \char`\"{}\+Insufficient Funds\char`\"{} error.

Example -- note this example assumes that the original charge was successful

\doxysubsubsection{\texorpdfstring{setSourceTransaction()}{setSourceTransaction()}}
{\footnotesize\ttfamily set\+Source\+Transaction (\begin{DoxyParamCaption}\item[{}]{\$value }\end{DoxyParamCaption})}

When creating separate charges and transfers, your platform can inadvertently attempt a transfer without having a sufficient available balance. Doing so raises an error and the transfer attempt fails. If you’re commonly experiencing this problem, you can use the {\ttfamily source\+\_\+transaction} parameter to tie a transfer to an existing charge. By using {\ttfamily source\+\_\+transaction}, the transfer request succeeds regardless of your available balance and the transfer itself only occurs once the charge’s funds become available.
