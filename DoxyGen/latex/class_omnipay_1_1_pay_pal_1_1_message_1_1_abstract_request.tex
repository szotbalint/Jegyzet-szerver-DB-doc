\doxysubsection{Detailed Description}
\mbox{\hyperlink{namespace_omnipay_1_1_pay_pal}{Pay\+Pal}} Abstract Request

This class forms the base class for \mbox{\hyperlink{namespace_omnipay_1_1_pay_pal}{Pay\+Pal}} Express Checkout and Pro Checkout requests. These are also known as \char`\"{}\+Payflow Gateway\char`\"{} requests and also \char`\"{}\+Pay\+Pal Classic APIs\char`\"{}.

According to the \mbox{\hyperlink{namespace_omnipay_1_1_pay_pal}{Pay\+Pal}} documentation\+:


\begin{DoxyItemize}
\item This is the recommended way to integrate when you want to accept payments with a completely customizable solution. This integration method leverages the \mbox{\hyperlink{namespace_omnipay_1_1_pay_pal}{Pay\+Pal}} Payflow Gateway to transmit payments your \mbox{\hyperlink{namespace_omnipay_1_1_pay_pal}{Pay\+Pal}} Internet Merchant Account; it also gives the merchant the flexibility to change payment processors without having to re-\/do their technical integration. When using \mbox{\hyperlink{namespace_omnipay_1_1_pay_pal}{Pay\+Pal}} Payments Pro (Payflow Edition) using Payflow Gateway integration, merchants can use Transparent Redirect feature to help manage PCI compliance.
\end{DoxyItemize}
