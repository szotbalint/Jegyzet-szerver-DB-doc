\href{http://travis-ci.org/composer/installers}{\texttt{ }}

This is for PHP package authors to require in their {\ttfamily composer.\+json}. It will install their package to the correct location based on the specified package type.

The goal of Installers is to be a simple package type to install path map. Users can also customize the install path per package and package authors can modify the package name upon installing.

Installers isn\textquotesingle{}t intended on replacing all custom installers. If your package requires special installation handling then by all means, create a custom installer to handle it.

If you\textquotesingle{}re a package author and need your package to be named differently when installed consider using the {\ttfamily installer-\/name} extra.

Due to the strict naming conventions, you as a package author actually need the package to be named and installed to {\ttfamily Plugin/\+Ftp}. Using the following config within your {\bfseries{package}} {\ttfamily composer.\+json} will allow this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{}
\DoxyCodeLine{    "{}name"{}: "{}shama/cakephp-\/ftp"{},}
\DoxyCodeLine{    "{}type"{}: "{}cakephp-\/plugin"{},}

\DoxyCodeLine{    \}}

What are they? The ability for a package author to determine where a package will be installed either through setting the path directly in their {\ttfamily composer.\+json} or through a dynamic package type\+: {\ttfamily \char`\"{}type\char`\"{}\+: \char`\"{}framework-\/install-\/here\char`\"{}}.

It has been proposed many times. Even implemented once early on and then removed. Installers won\textquotesingle{}t do this because it would allow a single package author to wipe out entire folders without the user\textquotesingle{}s consent. That user would then come here to yell at us.

Anyone still wanting this capability should consider requiring \href{https://github.com/oomphinc/composer-installers-extender}{\texttt{ https\+://github.\+com/oomphinc/composer-\/installers-\/extender}}.
