Faker provides adapters for Object-\/\+Relational and Object-\/\+Document Mappers (currently, \href{http://www.propelorm.org}{\texttt{ Propel}}, \href{http://docs.doctrine-project.org/projects/doctrine-orm/en/latest/}{\texttt{ Doctrine2}}, \href{http://cakephp.org}{\texttt{ Cake\+PHP}} and \href{https://github.com/mandango/mandango}{\texttt{ Mandango}} are supported). These adapters ease the population of databases through the Entity classes provided by an ORM library (or the population of document stores using Document classes provided by an ODM library).

To populate entities, create a new populator class (using a generator instance as parameter), then list the class and number of all the entities that must be generated. To launch the actual data population, call the {\ttfamily execute()} method.

Here is an example showing how to populate 5 {\ttfamily Author} and 10 {\ttfamily Book} objects\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<?php}
\DoxyCodeLine{\mbox{\hyperlink{test_2_documentor_8php_ab328ac5b606185b5e1706aa41380cc76}{\$generator}} = \mbox{\hyperlink{class_faker_1_1_factory_a0b3398649c2f374eb81301f7c4630ca0}{\(\backslash\)Faker\(\backslash\)Factory::create}}();}
\DoxyCodeLine{\$populator = \textcolor{keyword}{new} \mbox{\hyperlink{class_faker_1_1_o_r_m_1_1_propel_1_1_populator}{Faker\(\backslash\)ORM\(\backslash\)Propel\(\backslash\)Populator}}(\mbox{\hyperlink{test_2_documentor_8php_ab328ac5b606185b5e1706aa41380cc76}{\$generator}});}
\DoxyCodeLine{\$populator-\/>addEntity(\textcolor{stringliteral}{'Author'}, 5);}
\DoxyCodeLine{\$populator-\/>addEntity(\textcolor{stringliteral}{'Book'}, 10);}
\DoxyCodeLine{\$insertedPKs = \$populator-\/>execute();}

\end{DoxyCode}


The populator uses name and column type guessers to populate each column with relevant data. For instance, Faker populates a column named {\ttfamily first\+\_\+name} using the {\ttfamily first\+Name} formatter, and a column with a {\ttfamily TIMESTAMP} type using the {\ttfamily date\+Time} formatter. The resulting entities are therefore coherent. If Faker misinterprets a column name, you can still specify a custom closure to be used for populating a particular column, using the third argument to {\ttfamily add\+Entity()}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<?php}
\DoxyCodeLine{\$populator-\/>addEntity(\textcolor{stringliteral}{'Book'}, 5, array(}
\DoxyCodeLine{  \textcolor{stringliteral}{'ISBN'} => \textcolor{keyword}{function}() use (\mbox{\hyperlink{test_2_documentor_8php_ab328ac5b606185b5e1706aa41380cc76}{\$generator}}) \{ \textcolor{keywordflow}{return} \mbox{\hyperlink{test_2_documentor_8php_ab328ac5b606185b5e1706aa41380cc76}{\$generator}}-\/>ean13(); \}}
\DoxyCodeLine{));}

\end{DoxyCode}


In this example, Faker will guess a formatter for all columns except {\ttfamily ISBN}, for which the given anonymous function will be used.
