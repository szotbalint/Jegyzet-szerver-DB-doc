\doxysubsection{Detailed Description}
\mbox{\hyperlink{namespace_omnipay_1_1_pay_pal}{Pay\+Pal}} REST Complete Subscription Request

Use this call to execute an agreement after the buyer approves it.

Note\+: This request is only necessary for \mbox{\hyperlink{namespace_omnipay_1_1_pay_pal}{Pay\+Pal}} payments. Billing agreements for credit card payments execute automatically at the time of creation and so this request is not necessary for credit card payments.\hypertarget{class_omnipay_1_1_pay_pal_1_1_message_1_1_rest_complete_subscription_request_autotoc_md150}{}\doxyparagraph{Request Data}\label{class_omnipay_1_1_pay_pal_1_1_message_1_1_rest_complete_subscription_request_autotoc_md150}
Pass the token in the URI of a POST call to execute the subscription agreement after buyer approval. You can find the token in the execute link returned by the request to create a billing agreement.

No other data is required.\hypertarget{class_omnipay_1_1_pay_pal_1_1_message_1_1_rest_complete_subscription_request_autotoc_md151}{}\doxyparagraph{Example}\label{class_omnipay_1_1_pay_pal_1_1_message_1_1_rest_complete_subscription_request_autotoc_md151}
To create the agreement, see the code example in \mbox{\hyperlink{class_omnipay_1_1_pay_pal_1_1_message_1_1_rest_create_subscription_request}{Rest\+Create\+Subscription\+Request}}.

At the completion of a create\+Subscription call, the customer should be redirected to the redirect URL contained in \$response-\/$>$get\+Redirect\+Url(). Once the customer has approved the agreement and be returned to the return\+Url in the call. The return\+Url can contain the following code to complete the agreement\+:
